actions.json extension options

Cloud storage (S3)

Persists expenditure records and other extension data as JSONL objects in your own bucket. Leave empty to disable persistence — the live cost meter works either way.

Use an IAM key limited to this one bucket and prefix (PutObject + ListBucket only). The key is stored in extension local storage on this machine — scope it so that is an acceptable place for it to live.

OpenAI usage-read key (optional)

Enables the tracking-error readout (our estimates vs. actual charges). Create a restricted admin key with only the api.usage.read scope. Without it, everything else works and no OpenAI usage calls are made.
